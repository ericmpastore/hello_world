ShutdownOS

One command. Three languages. The same result.

EPastore · Apr 20 2026
Python 3.x
import os def main(): os.system( "shutdown /s /t 1" ) if __name__ == '__main__': main()

Uses Python's built-in os.system() to pass a shell command directly to the OS. The /s flag triggers a full shutdown and /t 1 sets a 1-second delay. Clean, minimal, and cross-shell friendly.

C++ C++11+
#include <cstdlib> int main() { system( "shutdown /s /t 1" ); return 0; }

Calls the C standard library's system() from <cstdlib> to invoke the shell command. A minimal main() demonstrates that low-level languages can interact with the OS in just a few lines of compiled code.

Java JDK 8+
import java.io.IOException; class ShutdownOS { public static void main (String[] args) { try { Runtime.getRuntime() .exec("shutdown /s /t 1"); } catch (IOException e) { e.printStackTrace(); } } }

Leverages Java's Runtime.getRuntime().exec() to spawn a native process from within the JVM. Wraps the call in a try/catch for proper IOException handling — idiomatic, safe, and Java-best-practice.

🔍

How it works — under the hood

Every implementation sends the Windows command shutdown /s /t 1 to the operating system shell. Despite different runtimes — an interpreter, a compiled binary, and a JVM — the end result is identical: a clean, graceful OS shutdown triggered after a 1-second countdown. A great reminder that the language changes; the system call stays the same.
