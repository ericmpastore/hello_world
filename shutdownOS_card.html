ShutdownOS

One command. Three languages. The same result.

EPastore · Apr 20 2026
Python 3.x
shutdown_os.py import os def main(): os.system( "shutdown /s /t 1" ) if __name__ == '__main__': main()

Uses Python's built-in os.system() to pass a shell command to the OS. /s triggers a full shutdown; /t 1 sets a 1-second delay. Clean and minimal.

C++ C++11+
shutdownOS.cpp #include <cstdlib> int main() { system( "shutdown /s /t 1" ); return 0; }

Calls C's standard system() from <cstdlib>. A minimal compiled binary that invokes the shell command directly — no runtime overhead.

Java JDK 8+
shutdownOS.java import java.io.IOException; class ShutdownOS { public static void main (String[] args) { try { Runtime.getRuntime() .exec("shutdown /s /t 1"); } catch (IOException e) { e.printStackTrace(); } } }

Uses Runtime.getRuntime().exec() to spawn a native process from within the JVM. Wrapped in a try/catch for proper IOException handling — idiomatic Java.

How it works — under the hood

Every implementation sends the Windows command shutdown /s /t 1 to the OS shell. Despite different runtimes — an interpreter, a compiled binary, and a JVM — the result is identical: a clean, graceful shutdown after a 1-second countdown. The language changes; the system call stays the same.
